## 1. 产品概述

美术馆导览预约系统是一套面向美术馆前台、讲解员和教育负责人的导览排班与预约管理页面。系统以展览日历为核心，围绕讲解场次组织观众预约，实现展厅限流、多语言场次管理、亲子/团体/无障碍场次跟踪，以及迟到记录与讲解反馈的全流程闭环。

- 目标用户：美术馆前台工作人员、讲解员、教育部门负责人
- 核心价值：减少排班冲突、提升观众入场效率、保障特殊群体服务覆盖率

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 前台工作人员 | 默认进入 | 查看展览日历、安排观众入场次、管理预约名单、记录迟到 |
| 讲解员 | 切换角色 | 查看本场观众结构与特殊需求、重点作品列表、提交讲解反馈 |
| 教育负责人 | 切换角色 | 查看亲子场/团体场/外语场占用统计、展厅容量总览、场次调整审批 |

### 2.2 功能模块

1. **导览总览页**：展览日历、当日场次看板、展厅限流状态、快捷操作入口
2. **场次管理页**：讲解场次列表、场次详情（观众结构/重点作品/集合点）、预约操作
3. **观众管理页**：观众名单、预约登记、儿童团队标记、无障碍需求、迟到记录
4. **统计总览页**：亲子场/团体场/外语场占用、展厅容量趋势、讲解反馈汇总

## 3. 核心流程

### 3.1 前台安排观众入场次的流程

前台工作人员在导览总览页查看当日场次看板，找到有空位的场次，在观众管理页登记观众信息（包括语言偏好、是否儿童团队、无障碍需求），系统自动匹配或手动分配到合适场次，确认集合点和时间。

```mermaid
flowchart TD
    "前台查看当日场次" --> "选择有空位场次"
    "选择有空位场次" --> "登记观众信息"
    "登记观众信息" --> "系统匹配场次"
    "系统匹配场次" --> "匹配成功?"
    "匹配成功?" --> "是: 确认预约"
    "匹配成功?" --> "否: 手动分配场次"
    "手动分配场次" --> "确认预约"
    "确认预约" --> "通知集合点与时间"
```

### 3.2 讲解员查看本场信息并提交反馈

讲解员切换到讲解员角色，查看本场观众结构（成人/儿童/团体/无障碍需求分布），确认重点作品清单，完成讲解后提交反馈。

```mermaid
flowchart TD
    "讲解员登录" --> "查看本场观众结构"
    "查看本场观众结构" --> "确认重点作品"
    "确认重点作品" --> "开始讲解"
    "开始讲解" --> "讲解完成"
    "讲解完成" --> "提交讲解反馈"
```

### 3.3 教育负责人监控场次占用

教育负责人查看亲子场/团体场/外语场的占用情况，发现容量不足或冲突时调整排班。

```mermaid
flowchart TD
    "教育负责人查看统计" --> "检查亲子场占用"
    "教育负责人查看统计" --> "检查团体场占用"
    "教育负责人查看统计" --> "检查外语场占用"
    "检查亲子场占用" --> "发现容量不足?"
    "检查团体场占用" --> "发现容量不足?"
    "检查外语场占用" --> "发现容量不足?"
    "发现容量不足?" --> "是: 调整排班"
    "发现容量不足?" --> "否: 继续监控"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：暖灰底色（#F5F0EB）搭配深炭灰文字（#2C2C2C），强调色使用美术馆金（#C9A96E）
- **辅助色**：限流预警红（#D94F4F）、亲子场暖橙（#E8915A）、外语场靛蓝（#4A7BA7）、团体场森绿（#5A9E6F）
- **按钮风格**：圆角微凸（border-radius: 8px），主操作金色填充，次操作描边
- **字体**：标题使用 Noto Serif SC（宋体气质），正文使用 Noto Sans SC
- **布局风格**：左侧固定侧栏导航 + 右侧内容区，卡片式模块分区，顶栏角色切换
- **图标风格**：线性图标（Lucide Icons），线宽1.5px，与文字等高
- **整体气质**：美术馆服务台——干净、有序、温润，信息层级清晰但不冰冷

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 导览总览页 | 展览日历 | 月历网格，有场次日期下方金色圆点标记，选中日期高亮金色底 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度 1280px
- 侧栏导航在 1024px 以下收起为图标模式
- 表格在窄屏下提供简化视图

## 5. 数据模型

### 5.1 核心数据实体

- **展览 Exhibition**：主题、起止日期、展厅列表
- **场次 Session**：关联展览、时间、语言、讲解员、类型（常规/亲子/团体）、集合点、容量上限、当前预约数
- **观众 Visitor**：姓名、电话、人数、预约场次、语言偏好、是否儿童团队、儿童年龄范围、无障碍需求类型
- **展厅 Gallery**：名称、容量上限、当前人数
- **讲解员 Guide**：姓名、语言能力、排班
- **迟到记录 LateRecord**：观众、场次、迟到时长
- **讲解反馈 Feedback**：场次、讲解员、评分、文字反馈、问题记录
